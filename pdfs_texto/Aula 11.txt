DML - Any
● Any
– Permitem outras comparações elemento-conjunto

--- PAGE 9 ---
DML - All
● ALL
– Permitem outras comparações elemento-conjunto
● Testa se um valor é >, <, =, … que TODOS os valores em um 
conjunto
● SELECT lista_atributos FROM tabela1
WHERE
atributo [<|>|=|!=|<>] ALL (conjunto | SQL)
– Não possui um mapeamento para a Álgebra Relacional
● Não é equivalente a operação de divisão

DML - Subconsulta
● Elemento
– Permite comparações elemento-valor
● Testa se um valor é igual ao valor retornado de uma consulta
SELECT lista_atributos FROM tabela1
WHERE
atributo [>,<,<>,!=,=,>=,<=] (consulta SQL)
● Pode ser usado como valor para uma coluna
SELECT lista_atributos, (subconsulta)
[FROM TABELA [WHERE condicao] ]

--- PAGE 13 ---
DML - Otimizações
● Otimização
– Podemos utilizar consultas aninhadas para realizar filtragens 
prévias nos dados
SELECT lista_atributos FROM (SUBCONSULTA)
WHERE condicao
– Ou mesmo para Junções
SELECT 
lista_atributos 
FROM (SUBCONSULTA_A) a 
JOIN (SUBCONSULTA_B) b ON 
a.pk = b.pk
[WHERE condicao]

Exercícios - IN
● Resolva o que se pede utilizando subconsultas [NOT] IN

Exercícios – ANY e/ou ALL
● Resolva o que se pede utilizando subconsultas ANY e/ou ALL
1) Números e andares de todos os ambulatórios, exceto o de menor 
capacidade
2) Nome e a idade do médico mais jovem (sem usar a função MIN!)
3) Nome e CPF dos pacientes com consultas marcadas para horários 
anteriores a todos os horários de consultas marcadas para o dia 
14/10/2016
4) Nome e CPF dos médicos que não atendem em ambulatórios com 
capacidade superior à capacidade de todos os ambulatórios do 
segundo andar
